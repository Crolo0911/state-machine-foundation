
# state-machine-foundation

## 项目概述

这是我第一个**完整、可运行**的 C++ 项目，也是我第一次有意识地使用**状态机（State Machine）思维**来组织系统结构。

项目形式上是一个基于控制台的学院管理系统（学生 / 老师 / 课程），但真正关注的并不是功能数量或界面表现，而是：

* 如何用**明确的状态**描述系统所处的位置
* 如何通过**状态切换**而非杂乱的条件判断来驱动程序流程
* 如何让系统结构更清晰、更容易理解与修改

这是一个以“结构与思维”为核心目标的项目，而不是以“完成度”为目标的项目。

---

## 为什么选择状态机

在本项目之前，我的程序逻辑更多依赖多层 `if / else` 与分散的流程控制，系统行为往往是隐式的、不直观的。

通过引入显式状态（如：主菜单、学生管理、老师管理、课程管理、退出），系统的运行路径变得可追踪、可推理，也更容易发现逻辑错误。

这个仓库，记录的是我**第一次真正体会到状态机在工程结构中价值的过程**。

---

## 项目定位说明

### 这个项目是：

* 一个**状态机思想的起点样本**
* 一个用于验证系统结构设计的实验性项目
* 一个保留“不完美”的学习记录

### 这个项目不是：

* 生产级管理系统
* 功能完整或体验导向的应用
* 使用复杂设计模式或框架的示例

本项目中存在的设计不足与人为 Bug 并未被全部“修到最优”，而是作为当前学习阶段的真实映射被保留下来。

---

## 架构说明（简述）

* 使用 `enum class` 明确定义系统状态与用户指令
* 主流程由状态切换驱动，而非深层嵌套逻辑
* 学生 / 老师 / 课程分别由各自的 Manager 负责管理
* 模块间尽量通过接口协作，避免直接控制彼此流程

整体目标是让“系统当前处于什么状态”这件事一眼可见。

---

## 关于版本与演进

本仓库将停留在**第一版（Foundation）**状态，不会继续演进为 V2。

后续项目将直接在更复杂、更贴近实战的场景中继续应用状态机思想，而不是在本项目中无限打磨细节。
